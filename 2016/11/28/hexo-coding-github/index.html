将hexo博客同时托管到github和coding

前言

最近发现，我之前把hexo托管在github上的访问速度太慢了，估计是因为github是国外的，访问速度才慢。所以想着部署一套在国内的托管平台。所以上网查找，看到胖逆的嘟嘟的博客:http://www.cnblogs.com/tengj/p/5352572.html, 有相关的介绍，根据博客的介绍，经了一番折腾，终于成功的把hexo的博客同时更新到github和coding上了。

Start

_config.yml配置
想要同时部署到2个平台，就要修改博客根目录下面的_config.yml文件中的deploy如下
根据Hexo官方文档需要修改成下面的形式

deploy:
  type: git
  message: [message]
  repo:
        github: <repository url>,[branch]
        gitcafe: <repository url>,[branch]

所以我的是这样：

deploy:
  type: git
  repo:
        github: git@github.com:sushaolian/sushaolian.github.io.git,master
        coding: git@git.coding.net:sushaolian/sushaolian.git,master

申请conding的账号

这是申请的网址：https://coding.net/

进入到网页中，点击注册，填写相关的信息。我这里已经注册成功的了，所以我就随便注册一个账号，方便演示给大家看。

注册成功之后，点击登录，创建一个新项目。

同步本地hexo到coding上
把获取到了ssh配置在上面的_config.yml文件中的deploy下，如果是第一次使用coding的话，需要设置SSH公钥，生成的方法可以参考coding帮助中心
如果你看过我的hexo在github的搭建，里面也有介绍如果如果生成，我这里直接使用之前部署github时已经生成的公钥。

本地打开 id_rsa.pub 文件，复制其中全部内容，填写到SSH_RSA公钥key下的一栏，公钥名称可以随意起名字。完成后点击“添加”，然后输入密码或动态码即可添加完成。

如果是和github用相同的公钥的话，那么请在.ssh文件夹中新建一个config文件，然后输入内容：

Host github.com
User 1243856760@qq.com (这里请填写你github的邮箱)
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443

添加后，在git bash命令输入：

ssh -T git@git.coding.net

如果得到下面提示就表示公钥添加成功了：

Coding.net Tips : [Hello ! You've conected to Coding.net by SSH successfully! ]

最后使用部署命令就能把博客同步到coding上面：

hexo deploy -g

如果在coding中，点击代码，如果能够看到上传的文件，那么就OK了。

pages服务方式部署

部署博客方式有两种，第一种就是pages服务的方式，也推荐这种方式，因为可以绑定域名，而第二种演示的方式必须升级会员才能绑定自定义域名。
我这里只介绍第一种，第二种方式请大家参考胖逆的嘟嘟的博客http://www.cnblogs.com/tengj/p/5352572.html，第二种要给钱的，所以这里就不推荐大家使用了。

第一种就是pages服务的方式
就是在source/需要创建一个空白文件，至于原因，是因为 coding.net需要这个文件来作为以静态文件部署的标志。就是说看到这个Staticfile就知道按照静态文件来发布。

cd source/
touch Staticfile  #名字必须是Staticfile

点击pages,部署分支填写master，因为我们的代码是提交到分支master上的，点击开启就可以了。

注意：

如果你的项目名称跟你coding的用户名一样，比如我的用户是叫susaolian,博客项目名也叫susaolian
那直接访问 susaolian.coding.me就能访问博客，否则就要带上项目名：susaolian.coding.me/项目名 才能访问
推荐项目名跟用户名一样，这样就可以省略项目名了

开启成功之后，就可以访问了。如果出现404的错误，那么就等10分钟左右。

个人域名绑定

我在万网上面购买了tengj.top域名，第一年只要2元哈哈，个人用的就不用com这种超级贵的域名了。现在要实现国内的走coding，海外的走github，只要配置2个CNAME就行。域名解析如下：

接着你们要在source文件中，新建一个文件CNAME，里面填写github上的博客地址。我的是sushaolian.github.io。然后把更新上去就可以了。

hexo deploy -g

接着在coding中，点击pages,填写绑定域名的地址，sushaolian.top

最后访问：sushaolian.top就可以看到自己的博客来了。
